# Research Materials

Everything in this directory is **read-only**: papers, sage scripts, and reference code that informed qBitcoin's design decisions but doesn't ship inside the binary. This is the audit trail. If you want to verify why a parameter is what it is, or reproduce a size/security claim from scratch, start here.

---

## Index

| Subdirectory | What's in it |
|---|---|
| [papers/](papers/) | Kudinov & Nick, *Hash-based Signature Schemes for Bitcoin* (IACR eprint 2025/2203, Revision 2025-12-05). PDF + extracted plain text. |
| [sage/](sage/) | The Blockstream Research SPHINCS-Parameters scripts (`security.sage`, `costs.sage`, `octopus_pmf.py`, original SPHINCS+ submission script) **plus our extensions** (`run_compact.sage`, `run_fallback.sage`, `probe_*.sage`, `dump_katfixtures.sage`). These are the authoritative oracles for our parameter choices and KAT fixtures. |
| [calibration/](calibration/) | `pqbc-cal` — a small standalone Go module that benchmarks SHA-256d on the host and proposes `Bits` for a given target block time. Used to derive `core.GenesisBits` and to sanity-check difficulty on dev machines. |
| [satoshi-v0.1/](satoshi-v0.1/) | Snapshot of Satoshi's original Bitcoin v0.1 source — `script.h`, `script.cpp`, `main.cpp`, `serialize.h`, `net.cpp`, etc. The opcode hex values in `script/opcodes.go` are derived verbatim from `script.h` here, and the v0.1 main flow informed several of our deviation decisions ("if Bitcoin v0.1 didn't have it, neither do we, unless the PQ swap forces it"). |
| [references/](references/) | A short link table — paper, sage repo, forum threads, Blockstream quantum page, related BIPs. |

The `+4 B` PORS+FP grind-salt difference between the paper oracle's `compute_size` and our `SigSize()` is documented in `parameters/signature-sizes.md`.

## Reproducing the parameter pins from scratch

You need [SageMath](https://www.sagemath.org/) installed (the scripts are SageMath 9+).

```sh
cd docs/research/sage

# Generate the canonical KAT fixtures (writes to crypto/hashsig/testdata/paper_kat.json):
sage dump_katfixtures.sage

# SHRIMPS compact-instance summary (security, size, sign/verify cost):
sage run_compact.sage

# SHRIMPS fallback-instance summary (paper's bold row + alt non-bold row):
sage run_fallback.sage

# Why we picked WOTS+C S=240 at l=32 instead of S=135 at l=18:
sage probe_shrincs_wotsc.sage

# Expected PORS+FP grind work for given (t, k, mmax):
sage probe_expwork.sage

# Full parameter table (regenerates the paper's Table 1 from first principles):
sage costs.sage --table 40       # filter to q_s = 2^40
sage costs.sage --table          # all q_s rows

# Single parameter set:
sage costs.sage --params W+C_P+FP 10 8 17 12 1 16 240
sage costs.sage --params W+C_P+FP 40 11 14 40 5 256 2040
```

Every `paper_kat.json` field is reproducible from `dump_katfixtures.sage` — that's its purpose.
